`@umpire/zustand` connects an umpire to a store slice. It works with vanilla Zustand stores and in any environment — no React required. The demo below is pure vanilla JS with small DOM helpers.

## Install

```bash
yarn add @umpire/core @umpire/zustand zustand
```

## `fromStore()`

```ts
function fromStore<
  S,
  F extends Record<string, FieldDef>,
  C extends Record<string, unknown> = Record<string, unknown>,
>(
  ump: Umpire<F, C>,
  store: {
    getState(): S
    subscribe(listener: (state: S, prevState: S) => void): () => void
  },
  options: {
    select: (state: S) => InputValues
    conditions?: (state: S) => C
  },
): UmpireStore<F>
```

## Return Surface

```ts
interface UmpireStore<F extends Record<string, FieldDef>> {
  field(name: keyof F & string): FieldAvailability
  get fouls(): Foul<F>[]
  getAvailability(): AvailabilityMap<F>
  subscribe(listener: (availability: AvailabilityMap<F>) => void): () => void
  destroy(): void
}
```

## Why Zustand Fits Well

Zustand's `subscribe()` already passes both `next` and `prev` state. That gives the adapter:

- previous values for `oneOf()` resolution
- previous and next snapshots for `flag()`

No extra effect system is needed.

## Example

```ts
import { createStore } from 'zustand/vanilla'
import { umpire, enabledWhen, requires } from '@umpire/core'
import { fromStore } from '@umpire/zustand'

// Define fields and rules — same as any umpire setup.
const fields = {
  email:       { required: true, isEmpty: (v: unknown) => !v },
  password:    { required: true, isEmpty: (v: unknown) => !v },
  companyName: { isEmpty: (v: unknown) => !v },
  companySize: { isEmpty: (v: unknown) => !v },
}

type Conditions = { plan: 'personal' | 'business' }

const signupUmp = umpire<typeof fields, Conditions>({
  fields,
  rules: [
    enabledWhen('companyName', (_v, cond) => cond.plan === 'business', {
      reason: 'business plan required',
    }),
    enabledWhen('companySize', (_v, cond) => cond.plan === 'business', {
      reason: 'business plan required',
    }),
    requires('companySize', 'companyName'),
  ],
})

// Create a vanilla Zustand store — your form state lives here.
const store = createStore(() => ({
  email: '', password: '', companyName: '', companySize: '',
  plan: 'personal' as Conditions['plan'],
}))

// Wire umpire to the store. select() picks field values;
// conditions() extracts external facts from the same store.
const umpStore = fromStore(signupUmp, store, {
  select: ({ email, password, companyName, companySize }) => ({
    email, password, companyName, companySize,
  }),
  conditions: (state) => ({ plan: state.plan }),
})

// Subscribe to availability changes — runs on every store update.
umpStore.subscribe((availability) => {
  console.log(availability.companyName.enabled)  // false (personal plan)
  console.log(availability.companyName.reason)    // 'business plan required'
  console.log(umpStore.fouls)                     // reset recommendations
})

// Mutate the store — umpire reacts automatically.
store.setState({ plan: 'business' })
// → companyName.enabled becomes true

// Clean up when done.
umpStore.destroy()
```

## Using with React

The adapter itself doesn't depend on React. If your app uses Zustand with React, bridge the `UmpireStore` using `useStore` from `zustand`:

```tsx
import { useStore } from 'zustand'

// Use the same vanilla store + fromStore setup from above.
// Then subscribe React components with selectors:
function CompanyField() {
  const companyName = useStore(store, (s) => s.companyName)
  const availability = umpStore.field('companyName')

  return (
    <input
      value={companyName}
      disabled={!availability.enabled}
      onChange={(e) => store.setState({ companyName: e.target.value })}
    />
  )
}
```

## Cleanup

Call `destroy()` when you no longer need the adapter so it unsubscribes from the underlying Zustand store.
